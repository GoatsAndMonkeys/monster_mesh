# Pocket Pikachu — V3 mono pet game

The V3 Pikachu pet — port of the 1998 Nintendo "Pocket Pikachu" pedometer
to a Heltec WiFi LoRa 32 V3 (ESP32-S3), running as a Meshtastic carousel
frame + MeshModule.

## Hardware

| Component | Detail |
|-----------|--------|
| Board     | Heltec WiFi LoRa 32 V3 (ESP32-S3) |
| Display   | 128×64 SSD1306 OLED — SDA=17 / SCL=18 / RST=21 |
| Button    | PRG (GPIO 0) |
| Pet area  | 72×60 px (36×30 at 2× scale), 2 px top padding |
| Sidebar   | 54 px wide, holds clock / hearts / watts / steps |

The pet's 72×60 game area sits at `GAME_X=0, GAME_Y=2`. The right-hand
sidebar starts at `SIDEBAR_X=74` and is rebuilt every frame from the live
`pikaStatus` struct (watts, friendship, NTP-synced clock).

## Game state machine

`game.cpp` keeps the canonical Pocket-Pikachu state machine. Walking +
play + bath + meal + sleep cycle through the day; the friendship counter
moves on each interaction and gates the four "stand" animation variants.

| State          | Frames |
|----------------|--------|
| `GS_STAND`     | `standBasic`, `standMad`, `standLike`, `standLove` per friendship tier |
| `GS_SLEEPING`  | breathing loop @ 1.2 s/frame |
| `GS_EATING`    | meal / snack sequence |
| `GS_BATH`      | scrub / dry |
| `GS_PLAYING`   | scheduled play activity |
| `GS_WALKING`   | shake-triggered walk burst |
| `GS_HAPPY`     | reaction after a gift |
| `GS_GIFT_MENU` | POW flash (600 ms) after confirming a gift |
| `GS_RAN_AWAY`  | terminal — friendship dropped below the floor |

The state machine has **no** WiFi or button polling — Meshtastic owns
those. Buttons are forwarded via `game_on_short_press()` /
`game_on_long_press()`; mesh packet RX increments steps via
`game_on_mesh_message()`.

## Animation system

- Each frame is 135 bytes (1080 pixels, 1-bit, MSB-first, row-major)
- Stored in PROGMEM; read via `frame_pixel(frame, index)`
- `AnimSeq` holds a `FramePtr[]` array, frame count, and per-frame delay
- All sequences live in the `Anims::` namespace
- `game_play_anim(&Anims::someName, loop)` starts an animation

The 45 frames are auto-generated by `convert_anims.py` from the original
web-emulator's `anims.json`. ANIM_MAP covers all 45 with no warnings.

## Pet mechanics

| Constant | Value | Meaning |
|----------|-------|---------|
| `SLEEP_HOUR` | 20 | Pikachu goes to sleep at 8 PM |
| `WAKE_HOUR`  | 8 | Wakes at 8 AM |
| `STEPS_PER_WATT` | 20 | Steps needed to earn 1 watt |
| `CONSECUTIVE_TO_WALK` | 20 | Shakes needed to trigger walk animation |
| `STEPS_UNLOCK1` | 15 000 | Tier 1 friendship unlock |
| `STEPS_UNLOCK2` | 30 000 | Tier 2 friendship unlock |
| `STEPS_GOAL` | 100 000 | Game completion |
| `SCREEN_TIMEOUT_MS` | 60 000 | Screen auto-off after 60 s inactivity |
| `ANIM_FRAME_MS` | 500 | Default ms between animation frames |
| `SLEEP_FRAME_MS` | 1200 | Slower breathing during sleep |

Friendship levels:

| Level     | Range       | Behaviour |
|-----------|-------------|-----------|
| Mad       | ≤ −500      | `standMad` |
| Ran away  | ≤ −1500     | Pikachu leaves, game over |
| Neutral   | 0           | `standBasic` |
| Likes     | ≥ 1500      | `standLike` |
| Loves     | ≥ 3000      | `standLove` |

## Step source

There's no IMU on the stock Heltec V3, so steps come from the mesh
instead: every received Meshtastic packet (broadcast or encrypted DM)
counts as one step via `game_on_mesh_message()`. This makes "walking the
dog" a social activity — your Pikachu levels up when the mesh is busy.

(A T190 / external IMU variant is planned. Wiring an LSM6DS3 to the
T190's SH1.0 QuickLink I²C connector would replace the mesh-packet
proxy with a real pedometer.)

## Re-generating animations.h

Download `anims.json` from the original Alberto-rp/Pocket-pikachu-emulator
repo, then:

```bash
python3 convert_anims.py path/to/anims.json
```

The `ANIM_MAP` in `convert_anims.py` has been corrected so all 45 frames
generate with no warnings.

## Carousel integration

`PocketPikachuModule::drawFrame` is registered as a Meshtastic carousel
frame alongside the node list / map / Pentest screens. Short press
advances the carousel; long press while focused on Pikachu opens the
internal menu (Clock → Status → back to game). The frame auto-closes
the menu if `drawFrame` isn't called for 2 s, in case the user navigated
away.
